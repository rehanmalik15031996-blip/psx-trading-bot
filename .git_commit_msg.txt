playbook: deterministic reactions overlay on every strategist run

After the May 11-13 sell-off the playbook engine fired the right cases
(imf_review_mission_week + brent_spike_e_and_p + mf_universe_distribution_broad)
but no machinery translated them into action changes; LLM was offline and
the fallback only emitted Phase-1 selected names as HOLD. This adds a
deterministic post-LLM/post-fallback layer so banks/cement/power get
auto-downgraded the moment a defensive case fires, regardless of the LLM.

Core engine
- brain/strategist_overlays.py: applies sector_overlay (downgrade_one /
  upgrade_one), symbol_overlay (min_bucket / max_bucket / weight_floor),
  cash_floor_pct, position_size_multiplier, conviction_cap. Also exposes
  compute_predictor_bias() for the rule predictor.
- brain/master_strategist.py: new _finalise_with_overlays() wired into
  all 4 exit paths (no-key, exception, unparseable, success). Enriched
  _fallback_decision to emit the full universe (so the overlay has actions
  to mutate). Writes data/_health/strategist.json badge.

Playbook content
- data/playbook/cases.json: added a reactions block to all 28 cases +
  3 new cases (risk_off_universe_session_pause as the mid-week safety net,
  brent_spike_cement_margin_squeeze, pre_imf_de_risk_window). Strengthened
  us_iran_oil_spike with event:us_iran_tension trigger.

Predictor
- scripts/generate_predictions.py: rule predictor now takes a playbook_bias
  dict (computed once per session) and adds sector + symbol score deltas
  before direction/action calculation.

UI / Health
- ui/app.py: three-state strategist banner (red LLM-offline-no-overlays,
  amber LLM-offline-overlays-salvaging, green-tint LLM-up-overlays-applied).
- scripts/_health.py: added 'strategist' workflow key.
- scripts/_write_strategist_today.py + _write_strategist_2026_05_13.py:
  apply overlay + write health badge after manual Cursor runs.
