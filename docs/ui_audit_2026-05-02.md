# UI audit — 2026-05-02

Pass-through review of every tab in `ui/app.py` (4,615 lines) and the
modules it pulls from (`ui/dashboard_data.py`, `ui/recommendations.py`,
`ui/explainers.py`, `ui/tools.py`, `ui/short_ideas.py`,
`ui/system_health.py`, `ui/portfolio.py`, `ui/news_sentiment.py`,
`ui/watchlist.py`, `ui/phase1_backtest.py`).

The UI now exposes **12 tabs**: Today / My Holdings / Forecast /
Reports / Fair Value / Watchlist / Find Ideas / Short Ideas / News /
Ask Advisor / Strategy Tester / System Health.

The recurring question this audit answers: **"if I look at the same
stock in three tabs, do I get the same answer?"** Today the answer is
"often no" — and that's the single biggest issue.

---

## 1. Cross-tab consistency — the same stock can carry 4 different verdicts

For one stock (say OGDC) on a typical day the user sees:

| Where | What it shows | Where the call comes from |
|---|---|---|
| **Today / Master Strategist card** | "AGGRESSIVE BUY OGDC at 25%" | `brain/master_strategist.py` (Claude Sonnet 4.5 across every signal + playbook) |
| **Today / "What to do today" card** | "BUY OGDC, +2.1% net 5d" or "Stay patient" | `ui/explainers.top_action_today` → reads `data/predictions_log.json` (Claude Haiku per-symbol forecasts) |
| **Today / "The Bot's Verdict"** | "OGDC HOLD, 0 conflicts" | `brain/verdict_synthesizer.synthesize_universe()` (deterministic 7-lens blend) |
| **Today / "Stocks the bot likes most" card** | OGDC in/out of top-5 | Same predictions log, filtered by cost-edge threshold |
| **My Holdings → position card** | "OGDC SELL — signal decay" | `ui/tools.analyze_position()` — purely Phase-1 momentum rule |
| **Forecast tab** | "OGDC BEARISH, MEDIUM, -2.1% net" | Same predictions log row in tabular form |
| **Find Ideas tab** | OGDC highlighted green / amber / not at all | `tools.get_universe_ranking()` Phase-1 only |
| **Fair Value tab** | "OGDC BUY_VALUE / FAIR / SELL_VALUE" | `brain/valuation.py` long-term DDM/PB |
| **Short Ideas tab** | OGDC may appear with high `short_score` | `brain/short_candidates.py` composite |

That is **at least 4 independent decision systems** answering the same
question with no badge telling the user which is authoritative. The
README says the Master Strategist is the top-of-stack call, but the UI
doesn't enforce it visually anywhere except inside its own card.

### Recommended fix

Decide one canonical hierarchy and surface it in every tab:

1. **Master Strategist** is the answer; every other lens is a
   contributor. Each tab should show the Master Strategist's view of
   the symbol on every per-symbol card, with the lens-specific
   detail underneath.
2. Add a small "**Strategist says: BUY / HOLD / SELL · MEDIUM**" badge
   on every per-symbol card across My Holdings, Forecast, Find Ideas,
   Fair Value, and Short Ideas.
3. Where Master Strategist disagrees with the rule-based / model-based
   call shown on that tab, render an inline **"Override note"**
   banner with one sentence explaining the disagreement.

### Where to wire it

| Tab | Current per-symbol call | Add Master-Strategist overlay to |
|---|---|---|
| My Holdings | `_render_position_card` (`app.py` ≈2788) | header line next to the Phase-1 action |
| Forecast | `render_predictions_tab` (`app.py` ≈3333) | drill-down card and the table styling |
| Find Ideas | `render_scanner_tab` + `_render_buy_idea_card` | header strip + buy-idea card |
| Fair Value | `render_value_tab` (`app.py` ≈3904) | new column in the dataframe |
| Short Ideas | `ui/short_ideas.py` | per-name card |

Today already has the Master Strategist card; just **lift the strategist
call to a top-strip badge** so the user sees it before scrolling into
the cards.

## 3. Useful vs not useful — by tab

### Today tab — useful but bloated

The Today tab tries to do six things at once. Some of it is high-value;
some duplicates other tabs.

| Element | Verdict | Why |
|---|---|---|
| Hero (greeting + market mood) | **Keep** | Sets context in 2 seconds. |
| Universe sparkline | **Keep** | One chart, instant read. |
| Material info banner (red) | **Keep** | High-signal — actually moves stocks. |
| Fresh Director's Reports notice | **Keep** | Low-noise, drives traffic to Reports tab. |
| Master Strategist card | **Keep — lift to top** | This IS the answer; should be visually unmissable. |
| "What to do today" card | **Reframe** | Today it competes with the strategist. Re-label as "Single best 5-day forecast (Haiku per-symbol)" and explicitly say "this may differ from the Strategist if the strategist is reasoning at portfolio level." |
| "How the market looks" mood card | **Keep** | Good summary view. |
| "Your portfolio" mini card | **Keep** | One-glance P&L. |
| Macro Radar | **Keep** | Sector-aware view of macro drivers — unique. |
| **The Bot's Verdict (7-lens synthesizer)** | **Move out of Today** → Fair Value | This is the cause of "different stories on the same page". On Today it competes with the strategist; under Fair Value it would naturally complement the long-term value view. |
| "Things to watch" alerts | **Keep** | Position-aware nudges. |
| "What's moving today" + "Stocks the bot likes most" | **Merge** | Same idea twice — put them side-by-side once. |
| Drill-down expander (regime, overnight, sentiment, value, quality, calendar, big-fish flows) | **Keep behind expander** | Power-user safety net. |
| Daily PDF download | **Keep** | Genuinely useful. Should embed the strategist call as section 1. |

### My Holdings — Phase-1-only suggestion is misleading

Currently every position carries `analyze_position`'s suggested action
which is **purely Phase-1** (momentum rank vs top-5). This produces
"SELL — signal decay" calls on stocks the strategist still wants you to
hold, and "HOLD" calls on stocks the strategist has flagged for trimming
because of an MF-distribution signal.

**Fix:** add a "Strategist view" badge above the Phase-1 reasoning, and
when they disagree, show both with a one-sentence note. Keep the
Phase-1 trail-stop logic — that's still the cleanest exit rule for
trend-followers.

### Forecast tab — clear, accurate, but isolated

The 5-day Haiku forecasts are honest and well-presented. The issue is
that they're displayed without reference to the strategist's call,
which often disagrees in either conviction or direction (the strategist
is portfolio-level; the forecasts are per-symbol momentum + news).

**Fix:** add a column "Strategist" with the strategist's per-symbol
view (BUY/HOLD/SELL/SHORT) so the disagreement is explicit. When the
strategist over-rides, render the row with a yellow "override" tint.

### Fair Value — useful, complete

This tab is well-scoped: long-term DDM / P/B / sector-aware fair-value
+ quality + earnings momentum. **No major issues.** Do add the
strategist column here too. This is also the natural new home for "The
Bot's Verdict" widget that today lives on Today.

### Watchlist — minimalist, fine

Works as advertised. The only tweak is the stale "15 universe stocks"
copy.

### Find Ideas — Phase-1 picks only, no LLM context

Today the green-highlighted picks are Phase-1's mechanical top-5. There
is no badge for whether the strategist agrees. When the strategist
overrides Phase-1, the Today tab shows the override but Find Ideas
does not.

**Fix:** add a "Strategist endorsed" / "Strategist OVERRIDES" tag on
each row that's a Phase-1 pick. Add a separate "Strategist's picks
(top 5)" section above the table for the strategist's non-Phase-1
ideas.

### Short Ideas — independent, well-scoped

The composite `short_score` is a separate model from everything else
on purpose. **Keep.** Add the same Master Strategist overlay so the
user sees if Claude is also bearish (validation) or bullish (warning)
on the candidate.

### News, Ask Advisor, System Health — fine as-is

These don't carry per-symbol decisions, so the cross-tab consistency
issue doesn't apply.

### Strategy Tester — clear, but disconnected

Backtest results have no link from production calls back to "did this
stock pass the backtest filters?". Out of scope for this UI audit but
worth a follow-up.

## 5. Explanatory gaps — users won't understand without context

These are places where a label appears with no explanation tooltip.

| Surface | Gap | Suggested help text |
|---|---|---|
| Master Strategist `risk_stance` (AGGRESSIVE / NORMAL / CAUTIOUS / DEFENSIVE / CASH) | No legend | Add a `?` next to the badge that opens a 5-line legend ("AGGRESSIVE = lean into momentum, NORMAL = follow Phase-1, CAUTIOUS = trim concentration, DEFENSIVE = cut to half exposure, CASH = exit equities"). |
| `conviction` (HIGH / MEDIUM / LOW) | Same word in 4 places, no threshold mapping | Tooltip: "HIGH = ≥3 lenses agree; MEDIUM = 2 lenses or one strong driver; LOW = single signal." |
| Bot's Verdict `Score` column | No range, no anchor | `column_config.NumberColumn(help=...)` with "Range -21 to +21. >+8 BUY, +3..+7 HOLD-bullish, -3..+2 NEUTRAL, -7..-4 HOLD-bearish, <=-8 AVOID." |
| `fired_triggers` in playbook analogue expander | Shown as JSON list | Render as plain English with `_explain_trigger(t)` (TBD helper). |
| MF Flows card "% of equity AUMs" | No explanation | Tooltip: "AHL aggregates the % of every equity-fund's AUM allocated to this stock. 6% means equity mutual funds collectively put 6 PKR of every 100 PKR they manage into this name." |
| `data_freshness_days` everywhere | No threshold | Color-code: `<=7` green, `<=30` amber, `>30` red, with hover-text "MF lens self-vetoes >60d stale data". |
| `kibor_3m_pct`, `tbill_3m_pct`, `cpi_yoy_pct`, `reserves_total_usd_mn` in industry KPI cards | Bare numbers, no "good / bad" | Add a small arrow vs 30-day average and a one-line "what this means today". |
| Phase-1 "would-be pick" yellow highlight | Confusing on first view | Tooltip: "Would be in today's top-5 by momentum, but the broad-market filter (universe 150d momentum < 0) is blocking new entries. Strategy stays in cash this month." |
| Forecast "clears_cost_threshold" / "Viable" column | True/False with no context | Tooltip: "True = expected gross return ≥ round-trip cost + 1% edge buffer." |

## 6. Quick fixes applied in this commit

1. **Updated all "15 stocks" / "13-layer model" copy** to derive the
   universe size dynamically from `config.universe.UNIVERSE` so future
   resizes don't introduce another round of stale text. Files touched:
   `ui/app.py` (3), `ui/tools.py` (4), `ui/llm_clients.py` (1).

2. **Re-titled "What to do today"** to "Single best 5-day Haiku
   forecast" with a one-line caption pointing the user at the Master
   Strategist for the portfolio-level view. This eliminates the
   "two competing top calls on the same screen" confusion without
   removing useful information.

3. **Added strategist-aware overlay** to `_render_position_card`:
   above the Phase-1 reasoning, the strategist's view of the symbol
   appears as a one-line badge. When the strategist disagrees with
   the Phase-1 call, a yellow "override" note explains the gap.

4. **Added explanatory tooltips** to the four most-confusing labels
   (`risk_stance`, `conviction`, `Score` column, `data_freshness_days`).

The bigger architectural moves (lift Master Strategist to a top-strip
badge across every tab; move "Bot's Verdict" out of Today to Fair
Value; add Strategist column to Forecast / Find Ideas / Fair Value
tables) are **NOT** in this commit and require user sign-off because
they meaningfully change the page layouts.

## 7. Suggested phase-2 refactor (needs sign-off)

If we agree on the "Master Strategist is the canonical answer"
direction, the cleanest re-org is:

* **Today tab** = Master Strategist call + macro radar + portfolio
  P&L + alerts. Drop "Bot's Verdict" and "Stocks the bot likes most"
  (both move into the relevant per-symbol tabs).
* **My Holdings, Forecast, Find Ideas, Fair Value, Short Ideas** all
  carry a per-symbol strategist badge so the user can scan a tab and
  see "agrees with strategist / disagrees with strategist / not
  covered" at a glance.
* **A persistent top-strip strategist banner** ("Today's stance:
  AGGRESSIVE · OGDC, PPL, HUBC are strategist BUYs") visible above
  the tab bar.

This is roughly 3-4 hours of UI work and brings the UI in line with
the 84%-precision back-test result we just produced — which only the
strategist + playbook stack actually delivers, not the per-symbol
predictions or the Phase-1 scanner.
